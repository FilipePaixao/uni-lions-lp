# Blueprint da Landing Page

## Objetivo

Conduzir um visitante qualificado de compreensão para confiança e, então, aplicação.

## Princípio

Cada seção precisa resolver uma pergunta. Se duas seções resolvem a mesma pergunta, consolidar.

## Estrutura

### 1. Header

**Pergunta:** onde estou e qual é o próximo passo?

- Logo Uni Lions.
- Navegação curta.
- CTA de aplicação.
- Sticky apenas se discreto e sem cobrir foco.

### 2. Hero

**Pergunta:** o que é e por que devo continuar?

Conteúdo:

- Eyebrow factual.
- Headline de decisão.
- Lead.
- CTA primário.
- CTA secundário.
- Prova rápida.
- Visual de autoridade.

Critério: entendimento em 10–15 segundos.

### 3. Prova factual

**Pergunta:** isso é programa real?

- 50h.
- São Paulo.
- Híbrido.
- Datas.
- Turma única.
- Aplicação.

### 4. Cenário

**Pergunta:** por que o método antigo não basta?

- Mudança de contexto.
- Limite de comando e controle.
- IA já entrou na operação.

### 5. Duas mãos

**Pergunta:** o que significa ambidestria?

- Defesa.
- Construção.
- Relação entre as duas.
- IA dentro do método.

Essa é a seção conceitual mais importante.

### 6. Resultado de capacidade

**Pergunta:** como vou decidir diferente?

- Visão sistêmica.
- Contexto.
- Governança.
- Priorização.
- Portfólio.

### 7. Entregáveis

**Pergunta:** o que levo?

Apresentar os oito artefatos e sua progressão para o Plano Diretor.

Evitar oito cards idênticos. Usar sequência, mapa ou timeline.

### 8. Jornada

**Pergunta:** como acontece?

- Antes.
- Durante.
- Banca.
- Formato.
- Datas.

### 9. Fit

**Pergunta:** é para mim?

Duas colunas:

- Faz sentido.
- Provavelmente não faz.

### 10. Sala

**Pergunta:** por que presencial e por aplicação?

- Pares.
- Curadoria.
- Ambiente.
- Sigilo.
- Limite real.

### 11. Quem conduz

**Pergunta:** confio nessas pessoas?

- Retratos reais.
- Bios curtas.
- Experiência relevante.
- Sem currículo enciclopédico.

### 12. Prova

**Pergunta:** alguém como eu valida isso?

- Depoimentos.
- Casos.
- Logos autorizados.
- Dados com fonte.

Se não houver prova validada, não fabricar. Substituir por método e transparência.

### 13. Investimento

**Pergunta:** qual o compromisso?

- Valor.
- Parcelamento.
- O que inclui.
- Comparação honesta.
- Termos antes do pagamento.

### 14. Aplicação

**Pergunta:** o que acontece quando clico?

- Aplica.
- Conversa e diagnóstico.
- Os dois lados decidem.

### 15. FAQ

Cobrir objeções restantes sem repetir toda a página.

### 16. Fechamento

- Missão.
- CTA.
- Privacidade.
- Termos.
- Contato.

## CTA map

| Local | CTA |
|---|---|
| Header | Aplicar para a turma |
| Hero | Aplicar para a turma |
| Entregáveis | Ver se faz sentido para minha empresa |
| Sala | Quero conhecer a turma |
| Investimento | Iniciar aplicação |
| Final | Aplicar para a turma de abertura |

Todos apontam para `/aplicacao`, salvo diagnóstico explicitamente intermediário.

## Densidade

- Hero: baixa.
- Cenário: média.
- Duas mãos: visual e conceitual.
- Entregáveis: alta, mas organizada.
- Investimento: objetiva.
- FAQ: progressiva.

## Critérios de remoção

Remover seção quando:

- não responde nova pergunta;
- só repete headline;
- contém afirmação sem prova;
- existe apenas para aumentar comprimento;
- depende de animação para fazer sentido;
- não contribui para aplicação qualificada.
