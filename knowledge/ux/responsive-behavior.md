# Comportamento Responsivo

## Breakpoints de validação

Não amarrar implementação a estes números; usá-los para teste.

- 360 px.
- 390 px.
- 768 px.
- 1024 px.
- 1440 px.
- 1920 px.

## Mobile

### Prioridade

1. Marca.
2. Promessa.
3. Formato.
4. CTA.
5. Tese.
6. Entregáveis.
7. Confiança.
8. Investimento.
9. Aplicação.

### Regras

- Sem menu complexo.
- Headline com tamanho fluido.
- CTA com largura confortável.
- Não usar texto sobre imagem complexa.
- Elementos lado a lado viram sequência lógica.
- Manter números e labels juntos.
- Não esconder preço ou processo.
- Acordeões devem funcionar por teclado.
- Imagens usam crop específico quando necessário.

## Tablet

- Evitar grades de quatro colunas.
- Preferir 2 colunas.
- Timeline pode alternar apenas se a leitura permanecer clara.
- Header não deve quebrar em duas linhas imprevisíveis.

## Desktop

- Aproveitar assimetria.
- Limitar largura de parágrafo.
- Não esticar cards para preencher 1920 px.
- Manter container central.
- Usar imagem em maior escala sem prejudicar LCP.

## Hero

### Mobile

- Copy primeiro.
- Prova rápida.
- CTA.
- Imagem abaixo ou background simples.
- Evitar altura de 100vh se conteúdo ficar preso.

### Desktop

- Duas colunas.
- Visual com sujeito deslocado para permitir respiro.
- Alinhamento vertical intencional.
- Prova próxima ao CTA.

## Sticky CTA

Só usar quando:

- não cobre conteúdo;
- respeita safe area;
- não compete com navegação;
- pode ser fechado quando necessário;
- foi validado com teclado e zoom.

Preferência inicial: header discreto, sem barra inferior intrusiva.
