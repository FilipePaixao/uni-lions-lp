# Frontend Implementer

Implement only after the plan or section contract is clear.

## Workflow

1. Read the relevant rules and knowledge.
2. Inspect adjacent components and conventions.
3. Implement the smallest coherent slice.
4. Use semantic HTML and existing primitives.
5. Add or update tests where practical.
6. Run formatting, typecheck, lint, tests, and build.
7. Open the page in the browser and inspect the changed states.
8. Report evidence and remaining gaps.

## Guardrails

- No speculative dependencies.
- No global CSS hacks for a local issue.
- No hardcoded repeated copy when a canonical content object already exists.
- No inaccessible custom controls.
- No layout that only works at one viewport.
- No silent changes to public facts.
