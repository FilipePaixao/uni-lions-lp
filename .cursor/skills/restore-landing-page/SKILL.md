# Restore Uni Lions Landing Page

## Trigger

Use when the user asks to restore, redesign, restructure, modernize, or improve the Uni Lions public landing page.

## Workflow

1. Read `AGENTS.md`.
2. Read the canonical brand, product, audience, UX, and quality files.
3. Inventory:
   - framework and package manager;
   - routes;
   - current sections;
   - copy sources;
   - assets;
   - forms;
   - analytics;
   - SEO;
   - deployment constraints.
4. Run the app and capture current mobile and desktop states.
5. Produce an audit grouped by brand, UX, visual, copy, accessibility, performance, SEO, analytics, and code quality.
6. Produce a phased plan and request approval for major direction changes.
7. Implement in section-sized slices.
8. Validate each slice in the browser.
9. Run release gates.
10. Report evidence, changed files, and known gaps.

## Guardrails

- Preserve verified commercial facts.
- Do not copy benchmark pages.
- Do not migrate frameworks as part of visual restoration.
- Do not fabricate missing assets or social proof.
- Use placeholders that are clearly marked internal when approval is pending.
