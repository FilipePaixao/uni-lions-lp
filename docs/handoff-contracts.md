# Contratos de Handoff entre Agentes

## Orchestrator → UX

Entregar:

- objetivo;
- fatos canônicos;
- limitações;
- audit atual;
- personas;
- rotas.

Receber:

- IA da página;
- perguntas por seção;
- CTA map;
- riscos.

## UX → Copy

Entregar:

- propósito de cada seção;
- objeção;
- espaço aproximado;
- prova disponível.

Receber:

- headline;
- support;
- CTA;
- claim source;
- pendências.

## Brand → Visual

Entregar:

- posicionamento;
- paleta;
- proibições;
- tese.

Receber:

- tokens;
- layout principles;
- image direction;
- motion guidance.

## Architecture → Implementation

Entregar:

- file plan;
- component contracts;
- content contracts;
- dependencies;
- test plan.

Receber:

- code;
- checks;
- changed files;
- known gaps.

## Implementation → QA

Entregar:

- route;
- feature scope;
- expected states;
- test data;
- known limitations.

Receber:

- defects;
- evidence;
- severity;
- retest.

## QA → Release Gate

Entregar:

- test matrix;
- screenshots;
- console/network;
- accessibility;
- unresolved issues.

Receber:

- GO/NO-GO;
- accepted risks;
- owners.
