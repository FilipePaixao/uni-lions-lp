# Prompts de Exemplo

## Auditoria inicial

```text
Use o agente unilions-orchestrator e o comando audit-landing.

Não altere arquivos. Leia a base canônica, rode o projeto, abra a landing em 390x844 e 1440x900 e produza uma auditoria com evidências. Separe fatos, hipóteses e pendências de negócio.
```

## Plano

```text
Com base na auditoria aprovada, use plan-restoration.

Quero preservar a stack e as rotas. Priorize clareza da proposta, experiência premium, mobile, acessibilidade e aplicação qualificada. Retorne fases, arquivos, riscos e critérios de aceite.
```

## Hero

```text
Implemente apenas header, hero e proof bar.

Use a paleta de knowledge/brand/visual-language.md. Não altere fatos. Não adicione dependências. Valide em 360, 390, 1024 e 1440 px. Rode os checks disponíveis e mostre evidências.
```

## Copy

```text
Use conversion-copywriter.

Reescreva a seção de entregáveis para mostrar progressão até o Plano Diretor. Não aumente a promessa. Cada claim precisa apontar para um arquivo canônico.
```

## Persona

```text
Use persona-validation e browser-qa.

Teste Ricardo, Mariana, Carlos, Ana e Eduardo. Não altere código. Retorne scorecard, dúvidas não respondidas, tempo de descoberta e top 5 mudanças.
```

## Release

```text
Use validate-release.

Execute todos os gates disponíveis. Não trate Lighthouse como única evidência. Retorne GO, GO WITH ACCEPTED RISKS ou NO-GO.
```
